# ADR-009: MCP server

狀態: accepted
日期: 2026-06-19

## 背景
hook 是被動攔截(coding AI 動了才擋)。MCP 讓 coding AI 能主動呼叫 supervisor
工具——開工前查進度、查契約、驗路徑——這是「先 hook 後 MCP」路線的第二步。

## 決策
`supervisor/mcp_server.py` 兩層分離:
- **工具邏輯層**:純函式(`tool_validate_event`/`tool_validate_diff_paths`/
  `tool_contract`/`tool_status`),回 dict,**不 import mcp**。直接複用 core/
  openapi_gate/progress,不重寫規則。→ 可單測,mcp 未裝也能 import。
- **server 綁定層**:`build_server()` 延遲 import FastMCP,註冊上述函式為 MCP
  工具;`main()` 跑 stdio。只有真要起 server 才需要 mcp 套件。

mcp 為選用依賴 `[mcp]`。

## 影響
- 新增 `supervisor/mcp_server.py`、選用依賴 `[mcp]`。
- 取捨:工具邏輯與綁定分離,多一層間接,但換來「核心可單測、mcp 可選」——
  符合「先 hook 後 MCP、能力沉澱不重寫」原則。
- 不在 MCP 層做新判斷;它只是 CLI 同核心的另一個入口。
