# ADR-008: OpenAPI 契約閘門

狀態: accepted
日期: 2026-06-19

## 背景
「code 照不照 spec」最可信的判法是把 spec 變成可機器檢查的契約。HTTP API 的
正式標準是 OpenAPI。需要一個確定性閘門,不靠 LLM。

## 決策
新增 `supervisor/openapi_gate.py`:
- `validate_openapi(path)` — 結構驗證(openapi/info/paths 鍵、每路徑的 operation
  需含 operationId)。結構壞 → BLOCK。
- `check_contract_coverage(spec, test_dir)` — 抽出 openapi 所有 operationId,掃
  契約測試目錄,每個 id 都須在某測試檔出現。缺 → BLOCK。
- `contract_gate(spec, test_dir)` — 合併兩者,回 GateDecision。

邊界:此閘門只做「契約有效 + 文件化端點有契約測試覆蓋」的確定性檢查。
**不做** runtime 一致性驗證(那需要跑 server + schemathesis,屬第二階段)。
取捨:覆蓋檢查是 traceability 級保證(有測試),非語意級;但它是確定性、可信、
能硬擋的,符合「可信才硬擋」原則。runtime 驗證未來再加。

## 影響
- 新增 `supervisor/openapi_gate.py`、CLI `contract` 命令。
- 範例 `contracts/openapi.yaml` + `tests/contract/`。
- 不依賴外部 OpenAPI 函式庫(只用 yaml + 字串掃描),保持輕量。
