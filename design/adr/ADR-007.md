# ADR-007: Codex 訂閱驗證後端

狀態: accepted
日期: 2026-06-19

## 背景
LLM 顧問層原本只接 anthropic API,要 key、付 per-token。使用者多半已有 Codex
訂閱,希望用它當驗證者腦,不再另付費。

## 決策
`review.py` 抽象出 backend 概念:
- `llm_review(...)` 維持為 anthropic 後端(不變)。
- 新增 `codex_review(...)`,shell 呼叫 `codex exec`,要求嚴格 JSON 輸出,解析成
  同一個 `ReviewResult`。
- 新增 `advisory_review(spec, design, diff, backend)` 統一入口,依 backend 分派。
- **兩後端共用同一套後處理規則**:注入篩檢先行、findings→REVIEW、confidence=0、
  缺席/失敗→ALLOW 降級。判決邏輯不因後端而異,只換「腦」。

邊界:Codex 後端仍把 spec/design/diff 當不可信輸入。判決永不單獨 ALLOW/BLOCK。

## 影響
- `supervisor/review.py`:加 codex_review + advisory_review 分派。
- `supervisor/cli.py`:`review --backend {anthropic,codex}`。
- 取捨:Codex 輸出非保證 JSON,需容錯解析;解析失敗走降級(ALLOW+註記),
  寧可漏報也不誤擋——符合「顧問不可信、確定性才硬擋」原則。
