# ADR-011: Provenance 產物來源證明

狀態: accepted
日期: 2026-06-19

## 背景
AI 產生的變更,需要的不只是「誰 merge」,而是「對應哪個 prompt/spec/diff/測試」。
單人專案不需 SLSA 加密簽章,但需要可追溯的純記錄。

## 決策
新增 `supervisor/provenance.py`:`build_attestation(base)` 用 git 蒐集 HEAD sha、
branch、變更檔、分支抽出的 SPEC/ADR 參照,加上 tasks 進度快照(複用 progress)、
時間戳,回 dict。CLI `attest` 寫 JSON。無 git 時欄位留空+標記,不崩。

邊界:純記錄,不簽章。SLSA/in-toto 加密證明列未來(需 OIDC/CI 環境)。

## 影響
- 新增 `supervisor/provenance.py`、CLI `attest`。
- 既有 `.github/workflows/ai-governor.yml` 已呼叫 `supervisor.cli attest`,現在真有了。
- 取捨:不簽章 = 可被竄改,但對單人本機追溯足夠;要防竄改再上 CI+OIDC。
