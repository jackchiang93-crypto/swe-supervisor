# ADR-010: Spec/Design 總覽(專案檔案夾)

狀態: accepted
日期: 2026-06-19

## 背景
SPEC 目前散在兩處(獨立檔 + current.md 章節),沒有單一入口看全貌。長對話時要回頭
看「做到哪、剩多少、要改哪條 spec」很麻煩。需要一個聚合視圖。

## 決策
新增 `supervisor/dossier.py`,聚合三來源(不重寫,複用 progress):
- **SPEC 蒐集**:掃 `specs/SPEC-*.md` 獨立檔;再從 `specs/current.md` 用標題正則
  `^#+\s*(SPEC-\d+)` 切出章節。兩來源合併、去重(獨立檔優先)。
- **ADR 關聯**:同編號 `design/adr/ADR-NNN.md`。
- **進度關聯**:`progress.load_status` 的 TaskStatus,以 spec 欄位對應。

提供三視圖:`spec_list()`(精簡表)、`spec_show(id)`(全文+ADR)、
`overview(full)`(統計+逐項;full 附全文)。CLI:`spec list`/`spec show`/`overview`。

## 影響
- 新增 `supervisor/dossier.py`、CLI `spec`(含 list/show 子命令)、`overview`。
- 取捨:current.md 章節解析靠標題正則,格式須一致(`# SPEC-NNN 標題`);獨立檔較穩。
  鼓勵新 SPEC 都用 `supervisor new` 生獨立檔。
- 純讀取聚合,不改任何狀態,不新增判斷邏輯。
