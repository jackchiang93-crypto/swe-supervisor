# ADR-012: Formal 形式驗證閘門(TLA+/Alloy)

狀態: accepted
日期: 2026-06-19

## 背景
高風險邏輯(狀態機/併發/一致性)值得形式規格。但 TLC model checking 需 java +
tla2tools.jar,不能假設環境都有。需要「有則跑、無則優雅降級」的閘門。

## 決策
新增 `supervisor/formal_gate.py`:`formal_gate(formal_dir)` 掃 *.tla/*.als。
- 無模型 → ALLOW + 附註(形式驗證為選配,不硬擋)。
- 有模型 + 偵測到 TLC(env `TLA_TOOLS` 指向 tla2tools.jar 且 java 存在)→ 跑
  `java -jar tla2tools.jar -tool` model check,fail → BLOCK,pass → ALLOW。
- 有模型但無工具鏈 → 降級:列出模型,標「工具鏈缺,略過實檢」(不假裝驗過)。

## 影響
- 新增 `supervisor/formal_gate.py`、CLI `formal`。
- 取捨:與 OpenAPI runtime 同哲學——真檢查需工具鏈,無則誠實降級,絕不假裝。
- 單人多半用不到;列為高風險模組專用,預設不擋。
